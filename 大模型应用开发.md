## 阶段二：异步编程（3-5 天）

大模型应用的核心是 I/O 密集型操作（API 调用），异步编程是必修课。

### 2.1 核心概念

```python
import asyncio

async def call_llm(prompt: str) -> str:
    # 模拟异步 API 调用
    await asyncio.sleep(1)
    return f"Response to: {prompt}"

async def main():
    # 并发调用多个大模型
    tasks = [
        call_llm("你好"),
        call_llm("天气如何"),
        call_llm("写一首诗"),
    ]
    results = await asyncio.gather(*tasks)
    for r in results:
        print(r)

asyncio.run(main())
```

### 2.2 重点掌握

- `async/await` 语法
- `asyncio.gather()` 并发执行
- `asyncio.create_task()` 任务管理
- `httpx` 异步 HTTP 客户端（替代 requests）

### 3.2 FastAPI 构建 API 服务

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class ChatRequest(BaseModel):
    prompt: str
    history: list[dict] = []

class ChatResponse(BaseModel):
    reply: str
    usage: dict

@app.post("/chat", response_model=ChatResponse)
async def chat(req: ChatRequest):
    # 调用大模型
    reply = await call_llm(req.prompt, req.history)
    return ChatResponse(reply=reply, usage={"tokens": 100})
```

重点掌握：

- Pydantic 数据校验与序列化
- 路由定义、请求/响应模型
- SSE（Server-Sent Events）实现流式输出
- 中间件（CORS、日志、认证）

## 阶段四：大模型 API 实战（1-2 周）

### 4.1 OpenAI API（通用范式）

```python
from openai import AsyncOpenAI

client = AsyncOpenAI(api_key="sk-xxx")

# 普通调用
response = await client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "你是一个助手"},
        {"role": "user", "content": "解释RAG是什么"},
    ],
    temperature=0.7,
)

# 流式调用
stream = await client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "讲一个故事"}],
    stream=True,
)
async for chunk in stream:
    print(chunk.choices[0].delta.content or "", end="")
```

### 4.2 必须理解的核心概念

| 概念             | 说明                                      |
| ---------------- | ----------------------------------------- |
| Prompt           | 提示词，与大模型交互的输入                |
| Token            | 大模型处理文本的最小单位                  |
| Temperature      | 控制输出随机性，0=确定性，1=创造性        |
| Context Window   | 上下文窗口，模型一次能处理的最大 token 数 |
| Function Calling | 让模型调用外部工具/函数                   |
| Streaming        | 流式输出，逐 token 返回                   |
| Multi-turn       | 多轮对话，通过 messages 数组维护上下文    |

## 阶段五：LangChain 框架（1-2 周）

> LangChain 是当前大模型应用开发最主流的编排框架。

### 5.1 核心模块

```
LangChain 核心架构：
├── Models      → 统一的大模型接口（ChatModel, Embeddings）
├── Prompts     → 提示词模板与管理
├── Chains      → 链式调用（将多个步骤串联）
├── Agents      → 智能体（模型自主决策调用工具）
├── Tools       → 工具定义（搜索、计算、数据库等）
├── Memory      → 对话记忆管理
└── Retrievers  → 检索器（RAG 核心）
```

### 5.2 快速上手示例

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 定义模型
llm = ChatOpenAI(model="gpt-4", temperature=0)

# 定义提示词模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个{role}专家"),
    ("user", "{question}"),
])

# 构建链
chain = prompt | llm | StrOutputParser()

# 调用
result = chain.invoke({"role": "Python", "question": "解释装饰器"})
```

### 5.3 LCEL（LangChain Expression Language）

```python
# 链式组合（管道语法）
chain = prompt | llm | output_parser

# 并行执行
from langchain_core.runnables import RunnableParallel
parallel = RunnableParallel(
    summary=summarize_chain,
    keywords=keyword_chain,
)

# 分支逻辑
from langchain_core.runnables import RunnableBranch
branch = RunnableBranch(
    (is_long_text, summarize_chain),
    (is_short_text, expand_chain),
    default_chain,
)
```

## 阶段六：RAG 检索增强生成（1-2 周）

> RAG 是企业级大模型应用的核心场景。

### 6.1 RAG 流程

```
文档 → 分块(Chunk) → 向量化(Embedding) → 存入向量数据库
                                              ↓
用户提问 → 问题向量化 → 相似度检索 → 取回相关文档片段
                                              ↓
                      大模型 ← 拼装 Prompt（问题 + 检索结果）
                        ↓
                     生成回答
```

### 6.2 核心代码

```python
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

# 1. 加载文档
loader = PyPDFLoader("docs/manual.pdf")
docs = loader.load()

# 2. 文档分块
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)

# 3. 向量化 & 存储
vectorstore = Chroma.from_documents(
    documents=chunks,
    embedding=OpenAIEmbeddings(model="text-embedding-3-small"),
)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

# 4. 构建 RAG 链
prompt = ChatPromptTemplate.from_template("""
根据以下上下文回答问题：
{context}

问题：{question}
""")

rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

answer = rag_chain.invoke("这个产品的保修期是多久？")
```

### 6.3 向量数据库选型

| 数据库   | 特点                  | 适用场景             |
| -------- | --------------------- | -------------------- |
| Chroma   | 轻量、本地运行        | 开发测试、小规模应用 |
| Milvus   | 高性能、分布式        | 生产环境、大规模数据 |
| Pinecone | 全托管云服务          | 快速上线、免运维     |
| Weaviate | 内置多模态支持        | 复杂查询场景         |
| FAISS    | Facebook 开源、纯本地 | 嵌入式场景、研究原型 |

## 阶段七：Agent 智能体开发（1-2 周）

### 7.1 Agent 核心思路

```
用户输入 → Agent（大模型）→ 决策：
  ├── 直接回答
  ├── 调用工具 A → 获取结果 → 再决策
  └── 调用工具 B → 获取结果 → 再决策
循环直到得到最终答案
```

### 7.2 自定义工具

```python
from langchain_core.tools import tool

@tool
def search_product(query: str) -> str:
    """搜索商品数据库，返回匹配的商品信息"""
    # 实际实现：查询数据库或 API
    return f"找到商品：{query}，价格：99元"

@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    return str(eval(expression))  # 生产环境请用安全的计算方式
```

### 7.3 构建 Agent

```python
from langchain_openai import ChatOpenAI
from langgraph.prebuilt import create_react_agent

llm = ChatOpenAI(model="gpt-4")
tools = [search_product, calculate]

agent = create_react_agent(llm, tools)

result = agent.invoke({
    "messages": [{"role": "user", "content": "帮我查一下手机的价格，然后打8折是多少"}]
})
```

### 7.4 LangGraph 工作流（进阶）

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated

class AgentState(TypedDict):
    messages: list
    next_action: str

# 定义节点
def classify(state: AgentState) -> AgentState:
    # 意图分类
    ...

def handle_qa(state: AgentState) -> AgentState:
    # 问答处理
    ...

def handle_task(state: AgentState) -> AgentState:
    # 任务执行
    ...

# 构建图
graph = StateGraph(AgentState)
graph.add_node("classify", classify)
graph.add_node("qa", handle_qa)
graph.add_node("task", handle_task)

graph.add_conditional_edges("classify", route_by_intent, {
    "qa": "qa",
    "task": "task",
})
graph.add_edge("qa", END)
graph.add_edge("task", END)

app = graph.compile()
```
