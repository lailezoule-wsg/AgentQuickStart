## 一、判断题（每题 2 分，共 20 分）

判断以下说法是否正确，正确写 `T`，错误写 `F`。

**1.** `None` 和空字符串 `""` 在布尔判断中结果相同。（T ）

**2.** `tuple` 一旦创建就不能修改其中的元素，所以 `t = (1, [2, 3]); t[1].append(4)` 会报错。（F ）

**3.** `dict` 的键可以是 `tuple` 类型。（F ）

**4.** `for i in range(3)` 和 `for i in range(0, 3)` 的效果完全相同。（T ）

**5.** `lambda x, y: x + y` 可以赋值给变量，也可以直接调用。（T ）

**6.** `try` 块中如果有 `return`，`finally` 块中的代码仍然会执行。（ T）

**7.** `set([1, 2, 2, 3, 3])` 的结果是 `{1, 2, 3}`，且元素顺序与原列表一致。（F）

**8.** `"{}".format(100)` 和 `f"{100}"` 的结果都是字符串 `"100"`。（T ）

**9.** `import math` 后，可以直接使用 `sqrt(4)` 而不需要加 `math.` 前缀。（F ）

**10.** `**kwargs` 接收的关键字参数在函数内部是一个字典。（T ）

**4.** 异常处理逻辑有误：

```python
try:
    x = int("abc")
except ValueError:
    print("转换失败")
    raise ## raise 会立即跳出 try-except 结构，所以第二个 except: 分支不会执行
except:
    print("未知错误")
finally:
    print("程序结束")
```

问题：这段代码在 `raise` 之后，`finally` 还会执行吗？`print("程序结束")` 会输出吗？说明理由并给出修改方案（如果不希望程序崩溃）。

`raise` 会重新抛出异常，`finally` **仍然会执行**，所以 `"程序结束"` 会打印。但之后程序会崩溃。修改方案：去掉 `raise`，或在 except 中处理完异常后不重新抛出。

### 题目 3：综合应用 - 配置解析器

编写函数 `parse_config(config_str, **defaults)`：

- 接收一个多行字符串 `config_str`，每行格式为 `key=value`
- `**defaults` 是默认配置项
- 返回一个字典，包含所有默认配置，并用 `config_str` 中的值覆盖同名默认项
- 自动将 value 转换类型：`"true"/"false"` → `bool`，纯数字 → `int`，带小数点 → `float`，其余保持 `str`
- 忽略空行和以 `#` 开头的注释行

```python
# 示例
config_text = """
# 数据库配置
host=localhost
port=5432
debug=true
max_connections=100
rate=0.75
"""

result = parse_config(config_text, host="127.0.0.1", port=3306, timeout=30)
# 期望返回：
# {
#     "host": "localhost",       # 被覆盖，str
#     "port": 5432,              # 被覆盖，int
#     "timeout": 30,             # 保留默认
#     "debug": True,             # 新增，bool
#     "max_connections": 100,    # 新增，int
#     "rate": 0.75,              # 新增，float
# }

def parse_config(config_str, **defaults):
    config1 = config_str.strip().splitlines()
    config_dict = {}
    for line in config1:
        if not line or line.startswith("="):
            continue
        if "=" in line:
            lineSplit = line.split("=")
            config_dict[lineSplit[0]] = defaults.get(lineSplit[0],lineSplit[1])
    return config_dict
```

### 改错题答案

**1.** `num = 0` 只是修改了循环变量，没有修改列表。正确写法：

```python
nums = [0 if n < 0 else n for n in nums]
# 或者用索引修改
for i in range(len(nums)):
    if nums[i] < 0:
        nums[i] = 0
```

**2.** 第一次访问不存在的键会 KeyError。修改：

```python
counts[char] = counts.get(char, 0) + 1
```

```python
for key, value in kwargs.items():
```

```python
def parse_config(config_str, **defaults):
    def convert(value):
        if value.lower() == "true":
            return True
        if value.lower() == "false":
            return False
        try:
            return int(value)
        except ValueError:
            pass
        try:
            return float(value)
        except ValueError:
            pass
        return value

    result = {**defaults}
    for line in config_str.strip().splitlines():
        line = line.strip()
        if not line or line.startswith("#"):
            continue
        key, value = line.split("=", 1)
        result[key.strip()] = convert(value.strip())
    return result
```
